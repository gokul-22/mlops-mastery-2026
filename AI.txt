The Full MLOps Picture
Here's how all these pieces connect in an end-to-end ML deployment:

You train a model and wrap it in a FastAPI app
You write a Dockerfile that packages the app
You push the Docker image to ECR (AWS's container registry)
You write a Kubernetes config telling EKS: "run 3 copies of this image, always"
EKS pulls the image from ECR and runs it across your worker nodes
AWS Load Balancer distributes traffic to all 3 copies
If traffic spikes, EKS auto-scales to 10 copies automatically
If a container crashes, EKS restarts it within seconds — zero downtime

So in short: Docker builds the container, ECR stores it, EKS runs and manages it at scale. That's the production MLOps deployment pipeline on AWS.
